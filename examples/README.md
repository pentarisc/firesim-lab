# firesim-lab examples

Two fully working target projects that demonstrate the firesim-lab framework.
Both are generated by `init-target` and then completed with real
implementations — use them as reference designs or starting points.

They are also compiled during `docker build` to pre-warm the Verilator
ccache, so your first `make verilator` on a real project is significantly
faster than a cold start.

---

## chisel-baremetal

**Bridges:** UART, FASED memory  
**Language:** Pure Chisel (no Verilog BlackBox)

A minimal bare-metal design written entirely in Chisel that:

1. Issues a single AXI4 read (1 beat, 8 bytes) from address `0x0`
2. Feeds the returned bytes into a Chisel-native UART 8N1 transmitter
3. Serialises them to the host via `UARTBridge`

### Run it

```bash
cd examples/chisel-baremetal
source env.sh
make compile
make verilator

# Loopback mode (no output file needed)
make run

# Write UART output to a file
make run ARGS="+uart-out=/tmp/chisel-baremetal.log"
cat /tmp/chisel-baremetal.log
```

### Pre-loading memory

The FASED model fills uninitialised memory with `0xDEADBEEF_DEADBEEF` by
default — good for a smoke test. To load a real payload, convert it to an
8-byte-aligned hex file and pass it at runtime:

```bash
# Create an 8-byte payload: "FireSim\n"
printf 'FireSim\n' | xxd -p | fold -w 2 | \
  awk 'NR==1{printf "@0\n"} {print}' > payload.hex

make run ARGS="+uart-out=/tmp/out.log +memloadhex=payload.hex"
cat /tmp/out.log   # → FireSim
```

### Key files

| File | Description |
|---|---|
| `src/main/scala/ChiselBaremetalTop.scala` | Top module + AXI4 FSM + UART TX |
| `src/main/scala/Configs.scala` | Target and platform configs |
| `src/main/cc/firesim_top.cc` | C++ simulation driver |

## verilog-blackbox

**Bridges:** UART, FASED memory  
**Language:** Chisel top + Verilog DUT via `BlackBox`

Demonstrates the Chisel `BlackBox` pattern: the Chisel top wires
`FASEDBridge` and `UARTBridge` to port bundles, and the actual RTL logic
lives in `AXIUARTPrinter.v` — a self-contained Verilog module that reads
from AXI4 and serialises bytes over UART.

This is the recommended starting point for teams whose existing RTL is
written in Verilog or SystemVerilog.

### Run it

```bash
cd examples/verilog-blackbox
source env.sh
make compile
make verilator
make run ARGS="+uart-out=/tmp/verilog-blackbox.log"
cat /tmp/verilog-blackbox.log
```

### Key files

| File | Description |
|---|---|
| `src/main/scala/VerilogBlackboxTop.scala` | Chisel top — wires DUT to bridges |
| `src/main/scala/AXIUARTPrinter.scala` | Chisel `BlackBox` declaration |
| `src/main/resources/vsrc/AXIUARTPrinter.v` | Verilog DUT implementation |
| `src/main/cc/firesim_top.cc` | C++ simulation driver |

## How the examples warm the Verilator ccache

During `docker build`, `examples/build-examples.sh` is called. It runs
`make compile` and `make verilator` on both examples inside the image.

```
docker build
  └── RUN build-examples.sh
        ├── make compile  (chisel-baremetal)  → GoldenGate Verilog
        ├── make verilator  (chisel-baremetal)  → g++ via ccache → binary
        ├── make compile  (verilog-blackbox)  → GoldenGate Verilog
        └── make verilator  (verilog-blackbox)  → g++ via ccache → binary
                                                   ↓
                                        ~/.cache/ccache/ now warm
```

When a user then runs `make verilator` on their own project, the Verilator-
generated C++ for common FireSim infrastructure files (simulation harness,
bridge drivers, FASED model, etc.) hits the cache — typically saving
10–20 minutes on the first compilation.

The generated binaries from the example builds are also left in
`examples/*/generated-src/` inside the image, so users can run them
immediately as a quick sanity check without compiling anything.

## Creating your own target

Use the generator script — it produces a project with the same structure as
these examples:

```bash
init-target --name my-design \
            --output-dir ~/projects \
            --bridge uart,fased \
            --feature verilog-blackbox

cd ~/projects/my-design
source env.sh
make compile
make verilator
make run
```

See the [main README](../README.md) for the full CLI reference.
